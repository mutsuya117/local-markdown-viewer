【概述】

Local Markdown Viewer 是一个Chrome扩展程序，可以以GitHub样式精美地显示本地Markdown（.md / .markdown）文件。
它会自动从标题（h1～h6）生成目录，并允许通过页内链接快速导航。
完全支持GitHub Flavored Markdown（GFM），包括表格、任务列表、语法高亮、Mermaid图表和数学公式显示等丰富功能。还配备了深色模式切换、打印功能和HTML导出功能，非常适合文档共享和分发。
无配置界面 - 安装即可使用的简洁设计。


【主要功能】

■ Markdown显示功能
・以GitHub样式显示，完全支持GitHub Flavored Markdown（GFM）
・支持140多种编程语言的语法高亮
・自动生成目录（h1～h6）、目录宽度可调整、响应式设计

■ Mermaid图表
・支持流程图、时序图、甘特图等多种图表类型
・在代码块中自动渲染

■ 数学公式显示（KaTeX）
・支持LaTeX格式的数学公式（行内和显示公式）
・一键切换开/关

■ 深色模式
・一键切换浅色/深色模式
・设置自动保存

■ 打印功能
・一键打印，目录和按钮自动隐藏
・保持当前模式（浅色/深色）

■ HTML导出功能
・一键下载为独立的HTML文件
・自动将图片嵌入为Base64（支持PNG、JPEG、GIF、WebP、SVG、BMP）
  - 本地图片：最大30MB/文件
  - 远程图片：最大20MB/文件（超过限制的保持为URL）
  - 并行处理快速下载（同时处理多个图片）
・无需扩展程序即可在任何浏览器中查看，非常适合文档共享
・近乎完全离线支持：图片和Mermaid图表完全离线，KaTeX公式也可基本显示

■ 多语言支持
・支持英语、日语、简体中文

■ 安全性
・隐私保护 - 文件内容不会发送到外部


【适用范围和限制】

■ 适用对象
・使用file://协议打开的Markdown（.md / .markdown）文件
・本地保存的Markdown文件

■ 不适用对象
・网页上的Markdown显示页面（GitHub、GitLab、Bitbucket等）
・远程Git仓库的预览页面
・其他文件格式（.txt、.html等）

■ 工作原理
・扩展程序仅在显示的页面内进行处理
・读取Markdown文本并转换为HTML进行显示
・不会将文件内容发送到外部服务器
・如果Markdown文件中有图片链接，则显示该图片
 （不会擅自访问未引用的文件）


【关于权限】

■ 必需设置
"允许访问文件网址"
启用此设置后，扩展程序可以读取本地文件（file://）。

■ 权限使用目的
・仅用于读取本地Markdown文件并转换为HTML格式进行显示
・在HTML导出功能中，用于读取Markdown中引用的图片文件并嵌入为Base64
・不会修改、删除或覆盖文件内容
・只读，仅用于显示

■ 关于隐私
・不收集数据 - 不收集用户数据或浏览历史记录
・不对外传输 - 不将文件内容发送到外部服务器
・无跟踪 - 不进行广告或访问分析
・完全本地处理 - 所有处理都在浏览器内完成


【适合使用此扩展程序的人】

・用Markdown编写README和技术规范文档，希望在浏览器中也能轻松阅读的人
・希望在本地安全查看不公开的内部或外部文档的人
・希望将Markdown文档作为精美的HTML文件进行分发和共享的人
・不想增加设置，优先考虑简洁性的人


【常见问题】

问：它适用于GitHub的Markdown页面吗？
答：不适用。此扩展程序仅格式化"本地打开的.md/.markdown文件"。
   不会干扰网页上的远程页面。

问：数据会被发送到哪里吗？
答：不会。它在页面内完成所有处理，不会向外部发送数据。
   所有处理都在浏览器内进行。

问：可以编辑文件吗？
答：不可以。此扩展程序是只读的。
   不会修改、删除或覆盖文件内容。

问：我不知道如何设置
答：安装时会自动打开设置指南页面，
   逐步说明设置方法。只需按照指南
   启用"允许访问文件网址"即可使用。

问：支持哪些语言？
答：支持英语、日语和简体中文。
   显示语言会根据Chrome浏览器的语言设置自动选择。
