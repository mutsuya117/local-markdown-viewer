# Markdown基本構文テスト

このファイルは、Markdownの基本構文が正しく表示されるかをテストします。

---

## 見出し（Headings）

# 見出しレベル1（H1）
## 見出しレベル2（H2）
### 見出しレベル3（H3）
#### 見出しレベル4（H4）
##### 見出しレベル5（H5）
###### 見出しレベル6（H6）

**期待される表示**:
- H1とH2は下線あり
- H1が最も大きく、H6が最も小さい
- すべての見出しが太字

---

## 段落（Paragraphs）

これは普通の段落です。複数の文を含むことができます。段落の後には空白行を入れます。

これは2つ目の段落です。段落と段落の間には空行が必要です。
改行だけでは新しい段落になりません。

段落内で改行したい場合は、
行末にスペースを2つ入れます。
これで改行されます。

**期待される表示**:
- 段落間に適切な余白がある
- 行末スペース2つで改行される

## 強調（Emphasis）

### 太字（Bold）

**太字のテキスト（アスタリスク2つ）**

__太字のテキスト（アンダースコア2つ）__

### 斜体（Italic）

*斜体のテキスト（アスタリスク1つ）*

_斜体のテキスト（アンダースコア1つ）_

### 太字+斜体

***太字かつ斜体（アスタリスク3つ）***

___太字かつ斜体（アンダースコア3つ）___

**期待される表示**:
- 太字：font-weight 600
- 斜体：font-style italic
- 太字+斜体：両方適用

## リスト（Lists）

### 番号なしリスト（Unordered List）

- 項目1
- 項目2
- 項目3
  - ネスト項目3-1
  - ネスト項目3-2
    - さらにネスト3-2-1
- 項目4

### 番号付きリスト（Ordered List）

1. 最初の項目
2. 2番目の項目
3. 3番目の項目
   1. ネスト項目3-1
   2. ネスト項目3-2
4. 4番目の項目

### 混合リスト

1. 番号付き項目1
   - 番号なし項目1-1
   - 番号なし項目1-2
2. 番号付き項目2
   1. 番号付きネスト2-1
   2. 番号付きネスト2-2

**期待される表示**:
- 適切なインデント
- 番号なしリストには・（箇条書き記号）
- 番号付きリストには1.、2.、3.

<https://www.google.com>

<test@example.com>

[見出しへのリンク](#見出しheadings)

**期待される表示**:
- 青色のリンク（#0969da）
- ホバー時に下線
- 外部リンクは新しいタブで開く

## 引用（Blockquotes）

> これは引用文です。
> 複数行にわたることができます。

> #### 引用内に見出し
>
> 引用の中に**太字**や*斜体*を含むこともできます。
>
> - リストも含められます
> - 2つ目の項目

### ネストした引用

> レベル1の引用
>
> > レベル2の引用（ネスト）
> >
> > > レベル3の引用

**期待される表示**:
- 左側に縦の線（0.25em、#d0d7de色）
- テキスト色がやや薄い（#57606a）
- ネストごとにインデント

## コード（Code）

### インラインコード

これは`inline code`の例です。`const x = 10;`のようなコードを文中に含めます。

### コードブロック（言語指定あり）

```javascript
function greet(name) {
  return `Hello, ${name}!`;
}
```

### コードブロック（言語指定なし）

```
plain text code block
no syntax highlighting
```

**期待される表示**:
- インラインコード：薄いグレー背景、等幅フォント
- コードブロック：薄いグレー背景、パディングあり、シンタックスハイライト

## 水平線（Horizontal Rules）

3つのハイフン:

---

3つのアスタリスク:

***

3つのアンダースコア:

___

**期待される表示**:
- 太い水平線（0.25em）
- グレー色（#d0d7de）
- 上下に余白（24px）

## 複合的な例

### リスト内に複数要素

1. **太字の項目**

   項目の説明文です。段落を含むことができます。

   ```python
   def example():
       return "コードブロックも含められます"
   ```

2. *斜体の項目*

   > 引用も含められます

   - ネストしたリスト
   - 2つ目

### 強調とリンクの組み合わせ

**[太字のリンク](https://example.com)**

*[斜体のリンク](https://example.com)*

`[コード内のリンク構文](https://example.com)`

## 総合チェックリスト

以下がすべて正しく表示されていることを確認してください：

- [ ] 見出し（H1〜H6）が正しいサイズと太さで表示
- [ ] H1とH2に下線がある
- [ ] 段落間に適切な余白
- [ ] 太字が太く表示される
- [ ] 斜体が斜めに表示される
- [ ] リストが適切にインデント
- [ ] リンクが青色で表示
- [ ] 引用が左側に縦線
- [ ] インラインコードに背景色
- [ ] コードブロックにシンタックスハイライト
- [ ] 水平線が太い線で表示
- [ ] ネストした要素が正しく表示

すべてチェックがついたら、基本構文のレンダリングは完璧です！
