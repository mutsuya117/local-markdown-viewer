# 段落テスト

このファイルは、段落が正しく表示されるかを詳しくテストします。

---

## テスト1: 基本的な段落

これは最初の段落です。

これは2番目の段落です。段落と段落の間には空行があります。

これは3番目の段落です。

**期待される表示**:
- ✅ 3つの段落が別々に表示される
- ✅ 段落間に16pxの余白がある
- ✅ 各段落が `<p>` タグで囲まれている

**F12で確認**:
```html
<p>これは最初の段落です。</p>
<p>これは2番目の段落です。段落と段落の間には空行があります。</p>
<p>これは3番目の段落です。</p>
```

## テスト2: 改行の処理

### パターンA: 空行なし（段落が分かれない）

1行目のテキスト
2行目のテキスト
3行目のテキスト

**期待される表示**:
- GFMでは `breaks: true` により、改行ごとに `<br>` が挿入される
- 1行目のテキスト<br>2行目のテキスト<br>3行目のテキスト

### パターンB: 空行あり（段落が分かれる）

1つ目の段落です。

2つ目の段落です。

3つ目の段落です。

**期待される表示**:
- 3つの独立した `<p>` タグになる
- 各段落間に余白がある

### パターンC: 行末スペース2つ（強制改行）

行末にスペース2つ
次の行
さらに次の行

**期待される表示**:
- 1つの `<p>` タグ内で `<br>` による改行

## テスト3: 長い段落

これは非常に長い段落のサンプルテキストです。段落は自動的に折り返されます。段落内に複数の文を含むことができます。段落の終わりまで、改行なしで続きます。段落と段落の間には空行を入れることで、新しい段落として認識されます。

これは2つ目の長い段落です。Markdownでは、段落の長さに制限はありません。適切な位置で空行を入れることで、読みやすい文章構造を作ることができます。

## デバッグ確認

F12デベロッパーツールのConsoleで以下を実行して確認：

### 段落の数を確認

```javascript
document.querySelectorAll('.markdown-body p').length
// → 段落の数が表示される
```

### 段落のmargin-bottomを確認

```javascript
const p = document.querySelector('.markdown-body p');
if (p) {
  const style = window.getComputedStyle(p);
  console.log('margin-bottom:', style.marginBottom);
  console.log('margin-top:', style.marginTop);
}
// → margin-bottom: 16px と表示されるはず
```

### br要素の数を確認（改行確認）

```javascript
document.querySelectorAll('.markdown-body br').length
// → <br>タグの数（breaks: trueで生成される）
```

---

## 期待される結果

| 項目 | 期待される動作 |
|------|---------------|
| 空行あり | 独立した `<p>` タグ、余白16px |
| 空行なし | `<br>` タグで改行（GFM） |
| 行末スペース2つ | `<br>` タグで改行 |
| リスト内段落 | インデントされた `<p>` タグ |
| 引用内段落 | 引用スタイル付き `<p>` タグ |

---

## トラブルシューティング

### 段落間に余白がない場合

- CSSの `.markdown-body p` のmargin-bottomが適用されているか確認
- 16pxの余白があるはず

### 改行が機能しない場合

- `breaks: true` の設定を確認
- `<br>` タグが生成されているか確認

### 段落が1つになってしまう場合

- Markdownファイルで段落間に**空行**があるか確認
- 空行がないと同じ段落として扱われる
