# 打ち消し線（Strikethrough）テスト

このファイルは、GitHub Flavored Markdownの打ち消し線機能をテストします。

---

## テスト1: 基本的な打ち消し線

~~これは打ち消された文字です~~

通常のテキストです。

~~この文全体が打ち消されています。~~

**期待される表示**:
- ✅ テキストに横線が引かれている
- ✅ `<del>` タグが使用されている
- ✅ 文字が読める程度の線の太さ

---

## テスト2: 文中に混在

これは普通のテキストで、~~ここは打ち消し~~、そして続きます。

最初の単語は~~削除~~訂正されました。

**期待される表示**:
- ✅ 打ち消し部分のみに線が引かれる
- ✅ 前後のテキストは通常表示

## テスト5: リスト内の打ち消し線

### 番号なしリスト

- ~~完了した項目1~~
- 進行中の項目2
- ~~キャンセルされた項目3~~
- 通常の項目4

### 番号付きリスト

1. ~~削除された手順1~~
2. 実施する手順2
3. ~~スキップされた手順3~~
4. 実施する手順4

### タスクリストと組み合わせ

- [x] ~~完了して削除されたタスク~~
- [x] 完了したタスク
- [ ] ~~キャンセルされた未完了タスク~~
- [ ] 通常の未完了タスク

**期待される表示**:
- ✅ リスト項目内で打ち消し線が機能
- ✅ チェックボックスと打ち消し線の組み合わせ

---

## テスト6: テーブル内の打ち消し線

| 項目 | 状態 | 備考 |

| 機能D | ~~廃止~~ | 使用しない |

- ✅ テーブルセル内で打ち消し線が機能

## デバッグ確認

F12デベロッパーツールのConsoleで確認：

```javascript
// 打ち消し線の数を確認
const delElements = document.querySelectorAll('.markdown-body del');
console.log('打ち消し線の数:', delElements.length);

// 打ち消し線のテキストを表示
delElements.forEach((del, i) => {
  console.log(`${i}: "${del.textContent}"`);
});
```

### HTML構造の確認

```javascript
// 最初の打ち消し線要素のHTMLを確認
const firstDel = document.querySelector('.markdown-body del');
if (firstDel) {
  console.log('HTML:', firstDel.outerHTML);
  const style = window.getComputedStyle(firstDel);
  console.log('text-decoration:', style.textDecoration);
}
```

**期待される出力**:
```
HTML: <del>これは打ち消された文字です</del>
text-decoration: line-through
```

## 期待される総合結果

| 場所 | 打ち消し線 | 表示 |
|------|-----------|------|
| 通常のテキスト | ✅ | 横線が引かれる |
| リスト内 | ✅ | 項目に横線 |
| テーブル内 | ✅ | セルに横線 |
| 引用内 | ✅ | 引用+横線 |
| 見出し内 | ✅ | 見出し+横線 |
| 太字と組み合わせ | ✅ | 太字+横線 |
| 斜体と組み合わせ | ✅ | 斜体+横線 |

すべて正しく表示されれば、打ち消し線機能は完璧です！

---

## 注意事項

GFM（GitHub Flavored Markdown）では、打ち消し線は `~~テキスト~~` の形式で記述します。

- ✅ 正しい: `~~text~~`
- ❌ 間違い: `~text~` （チルダ1つは打ち消し線にならない）
- ❌ 間違い: `<s>text</s>` （HTMLタグは使えない、DOMPurifyで削除される）
- ❌ 間違い: `<strike>text</strike>` （廃止されたタグ）
